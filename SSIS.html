I wrote a stored procedure that generates valid XML and inserts it into a table. There was too much data to write directly to a file so I insert the XML into an XML data-type column first and then BCP this XML to a file. I needed to devise C# code that could clean the file line by line due to the size of the data as I could not use C# XML extensions since the data could not fit into memory. The SQL script is thousands of lines long, due to the repetitive nature of FOR XML EXPLICIT. I decided that XML EXPLICIT was the most SQL Server-centric method to use for this specific project and would be sustainable without the ongoing purchase and maintenance of 3rd party tools. These records, tens of millions of lines of XML before shrinking, for hundreds of thousands of medical professionals, is ordered and cleaned up before the XML is validated against the industry standard MedBiquitous XSD file before zipping and sending via SFTP.
It was a fun challenge and I learned a lot while working on this project!
